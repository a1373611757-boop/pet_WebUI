Product Requirements Document

PRD - 邻宠到家

1. 产品概述

邻宠到家是一款面向城市养宠家庭的本地化上门宠物服务平台，提供喂养、遛狗、陪伴、洗护预约和订单跟踪。平台以“附近可信赖的人”和“透明可追踪的服务过程”为核心，降低宠物主人临时出行、加班、旅行时的照护焦虑。

2. 背景与机会

城市养宠人群增长，但宠物照护服务仍存在三个痛点：

  • 临时需求强，用户经常在出差、加班、旅行前才寻找照护。
  • 信任成本高，用户需要确认服务者身份、经验、评价和服务过程。
  • 供给管理难，平台需要平衡服务者空闲时间、地理覆盖、订单密度和履约质量。

3. 产品目标

  • 用户侧：让宠物主人在 3 分钟内完成可信服务选择与预约。
  • 服务侧：帮助服务者清楚管理日程、服务标准和收入。
  • 平台侧：通过数据看板监控转化、履约、渠道、服务结构与服务者利用率。

4. 目标用户

宠物主人

  • 年龄 24-40 岁，工作节奏较快，常有短时外出或加班场景。
  • 关注服务者可信度、宠物安全、过程反馈和价格透明。

邻里服务者

  • 有养宠经验、时间相对灵活，希望获取附近订单。
  • 需要清晰的订单说明、路线、服务记录和评价反馈。

运营人员

  • 关注获客效率、下单转化、服务履约、投诉风险和服务者供给。
  • 需要可扫描的数据看板支持日常复盘。

5. 核心场景

场景一：临时上门喂养

用户因周末短途出行，需要服务者每天上门喂猫、换水、铲砂并拍照反馈。

场景二：工作日晚间遛狗

用户临时加班，选择附近高评分服务者完成 30 分钟遛狗服务。

场景三：服务过程追踪

服务开始后，用户查看签到时间、定位、照片、服务备注和完成状态。

6. MVP 范围

用户端

  • 首页服务入口：喂养、遛狗、陪伴、洗护。
  • 服务者列表：评分、距离、价格、标签、可预约时间。
  • 服务者详情：介绍、经验、评价、服务能力、可服务宠物类型。
  • 预约下单：宠物信息、地址、服务时间、备注、价格确认。
  • 订单跟踪：待服务、服务中、已完成、取消。

服务者端

  • 接单与日程查看。
  • 服务签到、完成记录、图片反馈。
  • 收入与评价查看。

运营看板

  • GMV、订单量、转化率、复购率、履约率。
  • 订单趋势、预约漏斗、渠道贡献。
  • 服务类型分布、服务者利用率。
  • AI 洞察摘要与运营建议。

7. 非目标范围

  • 即时聊天 IM 系统。
  • 宠物医疗问诊。
  • 跨城市复杂调度。
  • 自动动态定价。

8. 指标定义

  • GMV：已支付订单总额。
  • 预约转化率：提交预约人数 / 浏览服务详情人数。
  • 履约率：已完成订单 / 已支付订单。
  • 复购率：周期内有 2 次及以上下单用户 / 下单用户。
  • 服务者利用率：已服务时长 / 可服务时长。

9. 风险与约束

  • 信任风险：需要实名认证、服务记录、评价体系和平台保障说明。
  • 履约风险：需要签到、定位、异常上报和取消规则。
  • 供给风险：需要关注不同区域和时间段的服务者可用性。
  • 宠物安全风险：需要宠物档案、服务禁忌、应急联系人和服务规范。

10. 成功标准

  • 新用户从首页到提交预约的路径清晰，核心流程少于 5 步。
  • 首批运营看板能支持每日复盘和渠道投放判断。
  • MVP 数据结构能覆盖订单、用户、服务者、服务类型和渠道来源。

# 邻宠到家｜经营数据看板与 AI 功能演示中心

> 本看板使用模拟经营数据和 mock AI 输出，仅用于作品集展示。不代表真实订单、真实收入、真实用户或真实模型调用结果。

## 定位

`dashboard/` 是邻宠到家作品集中的数据与 AI 展示模块。它回答两个面试官常问的问题：

1. 候选人是否能把业务链路拆成可衡量指标？
2. 候选人是否能把 AI 放进具体业务流程，而不是只做泛聊天入口？

## 首页模块

案例页建议按以下顺序讲解：

- 项目背景：社区宠物照护的信任和履约问题。
- 成果入口：PRD、小程序、H5、看板、AI 方案、宣传视频。
- 指标看板：预约、确认、收入、入住率、复购。
- AI 功能：客服、日报、运营文案、经营分析。
- 边界声明：模拟数据、mock AI、无真实后端和支付。

## 经营数据看板

看板不是为了展示图表数量，而是为了回答运营问题：

| 看板模块 | 回答的问题 |
| --- | --- |
| 指标卡 | 当前模拟业务规模和关键表现如何？ |
| 趋势图 | 访问、预约、确认、完成是否同步增长？ |
| 漏斗图 | 用户在哪一步流失明显？ |
| 渠道图 | 哪些渠道带来高质量确认订单？ |
| 服务结构 | 上门喂猫、遛狗、日托/寄养各自贡献如何？ |
| 入住率 | 周末或高峰时段是否存在供给压力？ |
| 复购分析 | 哪些服务更可能形成长期信任？ |

详细指标口径见：`../../docs/08-指标体系与数据看板.md`。

## AI 功能演示中心

当前包含四个 mock AI 模块：

- **AI 客服助手**：回答服务、价格、流程、安全边界等常见问题。
- **AI 寄养日报生成**：根据服务记录生成给宠物主人的日报草稿。
- **AI 运营文案生成**：为小红书、微信群、朋友圈生成可编辑文案。
- **AI 经营分析助手**：根据模拟看板数据输出“洞察 - 证据 - 建议”。

详细方案见：`docs/AI功能方案.md` 和 `../../docs/09-AI功能方案-作品集版.md`。

## AI 风险边界

- 不做宠物医疗诊断。
- 不承诺退款、赔付或平台规则外服务。
- 不自动发送未经人工确认的日报。
- 不自动发布运营文案。
- 不把模拟数据描述为真实经营结果。
- 高风险关键词需要转人工处理。

    components/ai/       # AI 功能演示组件
    data/                # mock 经营数据和指标计算

## 本地运行

```bash
npm install
npm run dev
```

## 部署说明

该项目可作为静态前端部署。部署前需确认：

- 页面中保留“模拟数据 / mock AI”声明。
- 不提交真实 API Key。
- 不把 public 中的展示文档描述为真实经营报告。

## 面试讲解重点

这部分体现的是数据分析和 AI 产品能力：我不仅能做图表，还能解释图表背后的业务问题；我不仅能写 AI Demo，还能定义输入、输出、Prompt、人工确认和风险边界。

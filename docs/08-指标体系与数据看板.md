# 08 指标体系与数据看板

> 使用 skills：`prd-development`、`product-strategy-session`。看板数据为 30 天模拟经营数据，仅用于展示指标设计和数据解释能力。

## 看板目标

数据看板用于回答运营问题，而不是堆叠图表：

- 哪些渠道带来访问，哪些渠道带来确认订单？
- 用户从访问到预约、确认、完成的漏斗是否顺畅？
- 哪类服务贡献订单和收入？
- 周末或节假日是否出现入住率压力？
- 用户是否有复购迹象？
- AI 是否能把指标变化转化为可执行建议？

## 核心指标口径

| 指标 | 口径 | 业务含义 |
| --- | --- | --- |
| 访问量 | 入口页面或看板模拟访问次数 | 衡量获客触达 |
| 服务点击 | 点击服务卡片或 CTA 的次数 | 衡量服务兴趣 |
| 预约量 | 提交预约意向数 | 衡量表单和信任转化 |
| 确认订单 | 通过人工确认的订单数 | 衡量价格、排期和服务者匹配 |
| 完成服务 | 已完成履约的订单数 | 衡量服务交付 |
| 预估收入 | 确认/完成订单对应的模拟收入 | 展示经营规模假设 |
| 入住率 | 日托/寄养容量使用率 | 衡量供给压力 |
| 复购率 | 重复下单用户占比 | 衡量信任沉淀 |
| 渠道确认转化率 | 确认订单 / 渠道访问 | 判断渠道质量 |

## 看板模块

- **指标卡**：预约量、确认订单、预估收入、入住率、复购率。
- **趋势图**：30 天访问、预约、确认和完成服务趋势。
- **漏斗图**：访问 -> 点击 -> 预约 -> 确认 -> 完成。
- **渠道图**：微信群、小红书、朋友圈、搜索、线下推荐等渠道贡献。
- **服务结构图**：上门喂猫、遛狗、日托、寄养等服务占比。
- **入住率趋势**：识别高峰日期和供给压力。
- **复购分析**：展示复购入口和用户留存假设。
- **AI 洞察卡**：用“洞察 - 证据 - 建议”解释业务动作。

## 业务解释示例

| 观察 | 可能解释 | 建议动作 |
| --- | --- | --- |
| 私域访问高但确认低 | 用户有兴趣，但价格/信任信息不足 | 优化照护者卡片和 FAQ |
| 周末入住率偏高 | 日托/寄养存在供给压力 | 提前预约提醒，增加周末照护者 |
| 上门喂猫复购高 | 猫咪上门服务标准化程度高 | 作为复购主推入口 |
| 小红书点击高但预约低 | 内容种草强，交易信任弱 | 增加真实流程说明和价格透明 |

## AI 经营分析原则

- 所有判断必须引用输入指标。
- 使用“可能”“建议验证”等表达，不做确定性经营结论。
- 建议必须可执行，例如优化某渠道内容、补充服务者、调整 CTA。
- 高影响动作需要人工确认。
- 明确标注数据为模拟数据。

## 面试展示重点

这套看板体现的是指标拆解和业务解释能力：候选人能从宠物照护行业问题出发，定义漏斗、渠道、服务结构、履约和复购指标，并把数据变化翻译成运营动作。

# 05 机会方案树

> 使用 skills：`opportunity-solution-tree`、`lean-ux-canvas`。本文件用于把业务目标拆成机会、方案和验证实验。

## Desired Outcome

提升宠物主人从“了解服务”到“提交预约”的信任转化，并让运营人员能用模拟数据解释服务表现。

可观察指标：

- H5/小程序入口点击率。
- 预约表单完成率。
- 订单确认率。
- 服务日报查看率。
- 复购率。

## Opportunity Solution Tree

### Opportunity 1：首次托付信任成本高

**问题**：用户不知道照护者是否可信，也担心宠物或家庭隐私风险。  
**方案**：

1. 照护者卡片突出片区、评分、服务次数、擅长类型。
2. FAQ 和信任说明解释实名认证、服务反馈、客服支持。
3. 首次短时试托作为低风险入口。

**验证实验**：用 H5 或小程序原型测试用户是否能在 1 分钟内说出“为什么这个照护者可信”。

### Opportunity 2：预约信息和价格不清晰

**问题**：临时需求下，用户不愿填写复杂表单，也担心价格不透明。  
**方案**：

1. 只保留服务类型、地址、日期、时间段、宠物信息、备注。
2. 使用本地价格规则实时计算预估价。
3. 明确标注最终价格会按特殊照护、节假日等确认。

**验证实验**：观察 5-8 名测试用户是否能独立完成预约表单并理解价格。

### Opportunity 3：服务过程缺少可追踪反馈

**问题**：用户不在现场时，只收到“已完成”无法降低焦虑。  
**方案**：

1. 订单详情展示状态、服务时间、宠物信息和价格。
2. AI 日报草稿根据服务记录生成结构化反馈。
3. 异常情况必须标记并转人工确认。

**验证实验**：给用户看两种反馈样例，比较“已完成”与“照片+状态+建议”的信任差异。

### Opportunity 4：运营看板不能转化为动作

**问题**：图表如果只展示数字，难体现产品/数据能力。  
**方案**：

1. 看板围绕渠道、漏斗、服务结构、入住率和复购组织。
2. AI 经营分析输出“洞察 - 证据 - 建议”。
3. 每个建议都标注人工确认事项和模拟数据边界。

**验证实验**：让面试官或同学看 3 分钟看板，复述当前最该优化的运营动作。

## Lean UX Canvas 摘要

- **Business Problem**：社区宠物服务的关键障碍不是线上化，而是信任、履约透明和运营复盘。
- **Business Outcomes**：提升预约提交、确认订单、日报查看和复购的指标表达能力。
- **Users**：宠物主人、社区照护者、运营人员、面试官。
- **User Outcomes**：宠物主人放心托付；运营人员快速看懂问题；面试官看见完整 PM 能力。
- **Solutions**：小程序 MVP、H5、数据看板、AI Demo、宣传视频。
- **Hypothesis**：如果把信任机制、预约链路、服务反馈和经营分析串成完整作品集，面试官能更快判断候选人的产品能力。
- **Learn First**：面试官是否能在 1 分钟内理解项目价值。
- **Least Work**：用 `docs/00-作品集总览.md` 和 README 先形成清晰入口。

# 10 优先级与路线图

> 使用 skills：`prioritization-advisor`、`roadmap-planning`。路线图面向作品集展示，不代表真实排期承诺。

## 优先级框架选择

当前项目是求职作品集 MVP，数据有限、目标是快速展示完整产品能力，因此采用轻量 ICE + 价值/努力判断，而不使用重 RICE。

- **Impact**：是否能明显提升面试官理解、用户信任或业务解释能力。
- **Confidence**：是否已有材料支撑，是否容易讲清楚。
- **Ease**：是否能在不改核心功能的情况下完成。

## Now / Next / Later

### Now：作品集标准化

- 建立 `docs/` 文档体系。
- 重写根 README、PRD、H5 README、看板 README、视频 README。
- 统一模拟数据和 mock AI 边界声明。
- 明确面试讲解路径。

### Next：展示体验增强

- 将 `01-项目介绍网页/README.md` 落地为静态入口页。
- 在 dashboard 中增加更明显的文档入口和边界声明。
- 为小程序关键流程补充截图或动图。
- 将 PRD、AI 方案、指标体系导出为可投递 PDF。

### Later：真实业务化探索

- 接入真实后端订单和权限。
- 接入真实 AI API 与日志、脱敏、费用控制。
- 增加服务者排班、地图服务半径、异常工单。
- 做真实用户访谈并更新 persona、JTBD 和旅程图。

## 不进入当前路线图

- 真实微信支付。
- 真实宠物医疗问诊。
- 即时聊天 IM。
- 跨城市复杂调度。
- 自动动态定价。
- 自动发布未经人工确认的 AI 内容。

## 沟通原则

路线图是作品集深化方向，不是交付承诺。每一项都应服务于“面试官能更快理解候选人能力”这个目标。
